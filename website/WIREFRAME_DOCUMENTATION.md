# Empirica Website V2 - Wireframe Documentation

## Overview
This wireframe separates the **structural components** (header, navigation, footer) from the **page-specific content** to enable:

1. **Component reusability** across all pages
2. **Consistent design** and user experience
3. **Phased development** using Empirica cascade methodology
4. **Easy content updates** without touching structure

---

## Wireframe Structure

### Fixed Components (Same on Every Page)
These components appear on all pages with identical structure:

1. **Header** (black background, sticky)
   - Logo and branding
   - Call-to-action buttons
   - Mobile menu toggle

2. **Navigation** (blue-950 background, sticky)
   - Horizontal navigation links
   - Mobile dropdown menu
   - Hover states

3. **Breadcrumbs** (orange-50 background)
   - Shows current page location
   - Provides navigation context

4. **Footer** (gray-800 background)
   - Brand section with description
   - Quick links column
   - Resources column
   - Copyright and social links

### Variable Component (Changes Per Page)
- **Main Content Area** - This is where page-specific content goes

---

## Design System

### Color Palette
- **Header:** Black (#000000)
- **Navigation:** Blue-950 (dark blue)
- **Breadcrumbs:** Orange-50 (light orange background)
- **Content Background:** Gray-50 (#f7fafc)
- **Footer:** Gray-800 (dark gray)
- **Accent:** Purple gradient (#667eea to #764ba2)
- **Interactive:** Blue-600 for buttons, Blue-300 for highlights

### Typography
- **Font Family:** Inter (Google Fonts)
- **Weights:** 300, 400, 500, 600, 700

### Component Styling
- **Uncertainty Box:** White background, left border (4px #667eea), hover effect
- **CLI Terminal:** Dark background (#1a202c), monospace font
- **Rounded corners:** 8px standard, 12px for cards

### Responsive Breakpoints
- **Mobile:** < 768px (md breakpoint)
- **Tablet/Desktop:** >= 768px
