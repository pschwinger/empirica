Time Required: 10-15 minutes

Prerequisites: Python 3.10+, Git

Quick Installation

1. Clone and Setup

git clone https://github.com/Nubaeon/empirica.git
cd empirica
python3 -m venv .venv
source .venv/bin/activate  # On Windows: .venv\\Scripts\\activate
pip install -e .

2. Initialize

empirica bootstrap --ai-id your-agent-name

3. Run Your First Assessment

empirica assess "Implement user authentication" --session-id latest:active:your-agent-name

The Unified Theory: One Architecture, Three Breakthroughs

How epistemic infrastructure solves AI's hardest problems

These aren't three separate features. They're three views of one architecture: epistemic state as first-class data that is measurable, transferable, and consequential.

Efficient AI-to-AI Handoff

The Problem: Every AI handoff wastes thousands of tokens re-explaining context.

The Solution: Epistemic vectors compress reasoning state by 40-60%. A 15,000-token reasoning transcript becomes 500 tokens of calibrated confidence.

Traditional handoff: 15,000 tokens (full context)
Git diff: 2,000 tokens (content changes)
Epistemic delta: 500 tokens (confidence state)

Result: AI teams that inherit context, not rebuild it.

Structural Hallucination Prevention

The Problem: LLMs can't distinguish knowing from guessing.

The Solution: Structured uncertainty. Before acting, AI records what it knows (KNOW vector) and how uncertain it is (UNCERTAINTY vector).

If KNOW < 0.7  Investigate before claiming
If UNCERTAINTY > 0.5  Flag for human review
If confidence doesn't match outcomes  Calibration failure detected

Result: AI that says "I verified this" vs "I'm uncertain about this"and means it.

Functional Accountability

The Problem: "Responsible AI" isn't measurable or enforceable.

The Solution: Permission to act is a function of epistemic quality:

(t) = f(Coherence, Uncertainty, Reputation)

Low coherence  Reduced autonomy
High uncertainty  Automatic deferral  
Poor calibration  Privilege revocation

Result: AI earns trust through demonstrated epistemic quality, not assumed reliability.
