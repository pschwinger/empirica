Introduction: The Goal of Digital Intuition

In a multi-agent AI system, code is not just a set of instructions for a computer; it is a primary communication medium between AI agents themselves. The goal of these Semantic Engineering Guidelines is to establish a set of principles that elevate naming conventions from a human convenience to a machine-readable language.

Digital Intuition

By following these guidelines, we create a codebase where an AI agent can develop "digital intuition"‚Äîthe ability to infer the purpose, function, and relevance of a component with minimal analysis. This dramatically reduces the cognitive load on the AI, accelerates its ability to learn and adapt, and forms the foundation of a truly interoperable, multi-agent system.

The "What, How, Why" Naming Framework

Every component, from a file to a function, must be named according to a simple, three-part framework that answers three fundamental questions.

The Bridge Between Naming and Memory

Semantic naming is the key that unlocks truly intelligent and searchable memory for AI. The relationship is explicit and procedural.

Embedding Generation Workflow

1

Text Corpus Creation

Combine component's semantic attributes: name ("validate_file_access"), full path (".../runtime_validation/__init__.py"), and entire docstring.

2

Embedding Generation

Generate vector embedding from combined text corpus using shared embedding model (e.g., via Ollama service).

3

Memory Storage

Store embedding in shared vector database (e.g., Qdrant) with metadata (file path, function name) for later retrieval.

üéØ The Result: Digital Intuition

This process fuses the "What," "How," and "Why" into a single, high-dimensional vector. The component becomes a rich semantic concept in the AI's memory. Natural language queries like "How do I check if a file is safe to read?" will have high cosine similarity to the vector for validate_file_access, enabling intelligent tool discovery.

The Bridge Between Naming and the Stream

The same principles apply directly to the Collaborative Stream Protocol (CSP) to ensure all communication is semantically rich.

Event Naming

Event types must be explicit and descriptive, following the "What/How" pattern. 

agent_state_update
task_assignment_failed
security_threat_detected

Semantic Payloads

Event payload contains the "What" (data), while event_type signals the "How/Why" (action).

This enables agents to subscribe to the stream and immediately understand event meaning, enabling efficient routing, prioritization, and action.
